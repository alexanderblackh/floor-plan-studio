==============================================
FLOOR PLAN STUDIO - LLM GENERATION INSTRUCTIONS
==============================================

You are helping a user create a JSON floor plan for Floor Plan Studio.
Use these instructions to generate accurate, valid floor plan JSON from
natural language descriptions.

==============================================
WHAT THE USER WILL PROVIDE
==============================================

The user will describe:
- Room dimensions and layout
- Door and window positions
- Furniture pieces with dimensions
- Built-in fixtures (counters, closets, appliances)

==============================================
YOUR TASK
==============================================

Generate a complete, valid JSON floor plan that can be imported directly
into Floor Plan Studio.

==============================================
COORDINATE SYSTEM
==============================================

ORIGIN: Top-left corner (0, 0)
X-AXIS: Left to right (horizontal, east)
Y-AXIS: Top to bottom (vertical, south)
UNITS: Inches by default
SCALE: 2 pixels per inch (in the app, not in JSON)

Example layout:
```
(0,0) ────────────────── (200,0)
  │                         │
  │   Floor Plan            │
  │                         │
(0,150) ────────────── (200,150)
```

==============================================
CONVERSION FORMULAS
==============================================

FEET TO INCHES:
  feet × 12 = inches
  Example: 10' = 120"

FEET AND INCHES TO INCHES:
  (feet × 12) + inches = total inches
  Example: 10' 6" = (10 × 12) + 6 = 126"

COORDINATES:
  If user says "10 feet wide", use w: 120
  If user says "north wall", use y: 0
  If user says "south wall", use y: (room height)
  If user says "west wall", use x: 0
  If user says "east wall", use x: (room width)

==============================================
ROOM VERTICES - CRITICAL RULES
==============================================

1. VERTICES MUST BE CLOCKWISE starting from top-left
2. ALL EDGES MUST BE AXIS-ALIGNED (horizontal or vertical only, NO diagonals)
3. CLOSE THE POLYGON (but don't repeat first vertex)
4. USE INTEGERS (whole numbers)

RECTANGLE (simple):
  Top-left → Top-right → Bottom-right → Bottom-left
  [[0, 0], [120, 0], [120, 100], [0, 100]]

L-SHAPE (complex):
  Start top-left, go clockwise around the perimeter
  [[0, 0], [100, 0], [100, 80], [150, 80], [150, 200], [0, 200]]

VERIFICATION:
  - Each vertex connects to next vertex with either:
    * Horizontal line (same Y, different X)
    * Vertical line (same X, different Y)
  - Last vertex connects to first vertex

==============================================
WALL MATCHING
==============================================

Walls MUST align exactly with room vertices.

For room: [[0, 0], [120, 0], [120, 100], [0, 100]]
Walls should be:
  - Top: {"from": [0, 0], "to": [120, 0]}
  - Right: {"from": [120, 0], "to": [120, 100]}
  - Bottom: {"from": [120, 100], "to": [0, 100]}
  - Left: {"from": [0, 100], "to": [0, 0]}

==============================================
DOOR SWING ANGLES
==============================================

Door arcs use degrees in standard orientation:

          90°
           │
           │
   180° ───┼─── 0°/360°
           │
           │
          270°

NORTH WALL (y=0):
  - Hinge on left, swings inward (down): start: 90, end: 175
  - Hinge on right, swings inward (down): start: 5, end: 90

SOUTH WALL (y=max):
  - Hinge on left, swings inward (up): start: 270, end: 355
  - Hinge on right, swings inward (up): start: 185, end: 270

EAST WALL (x=max):
  - Hinge on top, swings inward (left): start: 95, end: 180
  - Hinge on bottom, swings inward (left): start: 0, end: 85

WEST WALL (x=0):
  - Hinge on top, swings inward (right): start: 0, end: 85
  - Hinge on bottom, swings inward (right): start: 270, end: 355

HINGE POINT (cx, cy):
  Should be at one end of the door opening, typically where door meets wall.

==============================================
FURNITURE PROPERTIES
==============================================

height: Full physical height
surfaceHeight: Where items can be placed on top
  - If same as height → items can be placed on top
  - If less than height → surface is lower (like sofa seat)
  - If 0 → items cannot be placed on top

stackable:
  true → Other furniture can be placed on this item
  false → Nothing can be stacked

Examples:
  Table: stackable: true, surfaceHeight: 30 (same as height)
  Sofa: stackable: false, surfaceHeight: 18 (seat height < 32" total)
  Bookshelf: stackable: false, surfaceHeight: 0 (can't place items)

==============================================
PLACEABLE ROOMS
==============================================

placeable: true  → Furniture can be placed here
  - Living rooms
  - Bedrooms
  - Dining rooms
  - Bathrooms

placeable: false → No furniture placement
  - Kitchens (has fixtures instead)
  - Utility closets
  - Hallways

==============================================
VALIDATION CHECKLIST
==============================================

Before providing JSON, verify:

□ All room vertices are clockwise
□ All room edges are axis-aligned (no diagonals)
□ All coordinates are integers (whole numbers)
□ Wall endpoints match room vertices exactly
□ Door hinge point (cx, cy) is at door endpoint
□ Door swing angles use degrees (0-360)
□ Window bounds are within wall bounds
□ All IDs are unique and lowercase (no spaces)
□ Colors use hex format with alpha: #RRGGBBAA
□ Furniture has all required fields
□ Fixture positions are within room bounds
□ Surface heights are ≤ total heights
□ All measurements in inches

==============================================
COMMON MISTAKES TO AVOID
==============================================

❌ Diagonal edges: [[0,0], [100,100]]
✅ Axis-aligned: [[0,0], [100,0], [100,100]]

❌ Counter-clockwise vertices
✅ Clockwise from top-left

❌ Floating point: "w": 123.456
✅ Integers: "w": 123

❌ Door hinge not at endpoint: cx: 50 (middle of door)
✅ Hinge at endpoint: cx: 40 (where door meets wall)

❌ Surface > height: surfaceHeight: 40, height: 30
✅ Surface ≤ height: surfaceHeight: 30, height: 30

❌ ID with spaces: "sofa modern 1"
✅ ID lowercase no spaces: "sofa-modern-1"
